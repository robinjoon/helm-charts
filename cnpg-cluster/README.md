# cnpg-cluster

A single shared **CloudNativePG** PostgreSQL cluster for the homelab, plus
per-app databases and owner roles. Replaces the deprecated Bitnami-legacy
PostgreSQL chart.

## Prerequisites

The CloudNativePG **operator** must be installed first (separately):

```sh
helm repo add cnpg https://cloudnative-pg.github.io/charts
helm install cnpg-operator cnpg/cloudnative-pg -n cnpg-system --create-namespace
```

(In this homelab it is managed by the Argo CD `cnpg-operator` Application.)

## What this chart creates

- one `Cluster` (`instances: 1`, small resources, k3s `local-path` storage)
- for each entry in `.Values.databases`: a login **role** + a **database** it owns

## Secrets (never in Git)

- The cluster's superuser/app credentials are **auto-generated by CNPG**
  (Secret `<cluster>-app`, e.g. `pg-app`).
- Each app role's password is read from an **out-of-band** Secret named
  `cnpg-role-<app>` (type `kubernetes.io/basic-auth`). Create it manually:

  ```sh
  kubectl -n <namespace> create secret generic cnpg-role-n8n \
    --type kubernetes.io/basic-auth \
    --from-literal=username=n8n \
    --from-literal=password="$(openssl rand -base64 24)"
  ```

## Adding an app

1. Add it to `values.yaml`:

   ```yaml
   databases:
     - name: n8n
   ```

2. Create the `cnpg-role-n8n` Secret (command above).
3. Sync. The app connects to host `pg-rw.<namespace>.svc` (read/write service)
   as user `n8n`, database `n8n`.
